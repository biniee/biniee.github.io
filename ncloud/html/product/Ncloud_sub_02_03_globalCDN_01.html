특징

세계 어느 곳에서나 다양한 콘텐츠를 빠르고 안전하게 전달

전 세계 120여 개 국가에 분산 배치된 수십만 대의 캐시 서버를 통해 해외 사용자에게 빠르게 콘텐츠를 전송할 수 있습니다.
동영상 및 음악 스트리밍, 대용량 파일 다운로드 등으로 인터넷 상의 트래픽이 증가했을 때
캐시 서버로 원본 서버의 부하를 감소시켜 콘텐츠 전송 품질을 높이고 속도도 향상시킬 수 있습니다.
(Global CDN은 해외 거점을 기반으로 한 상품이므로, 국내에서 서비스를 제공해야 할 경우에는 CDN 상품을 이용해주시기 바랍니다.)

  • 글로벌 서비스의 연속성 확보

    해외 사용자에게 가장 인접한 서버를 통해 콘텐츠를 전송하기 때문에 빠르고 안정적입니다. 사용자 위치에서 가까운 서버에 장애가 발생하면 또다른 근접 서버로 우회하여 가용성을 높입니다. 트래픽이 증가할 시에는 고도로 분산된 캐시 서버를 통해 원본 서버를 보호하고, 대용량의 트래픽을 처리할 수 있습니다.

  • 간단한 신청 및 활용

    웹 기반의 콘솔 환경에서 간단하게 신청하고 설정할 수 있습니다. 콘솔에서 설정한 값을 전 세계 서버에 배포하여 빠르고 편리하게 서비스를 이용할 수 있습니다. 캐싱, 압축 등 다양한 옵션을 제공하여 서비스 성격에 따라 맞춤 구성이 가능합니다.

  • 다양한 상품과의 연계 가능

    네이버 클라우드 플랫폼의 다른 상품과 연동하여 사용할 수 있습니다. 기존에 사용하고 있던 File Storage와 Server 상품을 CDN의 원본 서버 저장소로 활용할 수 있으며, Live Transcoder 이용 고객은 Global CDN 상품과의 연동을 통해 전 세계 사용자들에게 안정적인 생중계 서비스를 제공할 수 있습니다.

  • 효율적인 비용 관리

    Global CDN의 이용 금액은 콘텐츠 전송량에 비례하여 사용한 만큼만 과금되기 때문에 비용을 예측할 수 있어 효과적인 관리가 가능합니다. 별도의 인프라 구축 비용 없이도 수십만 대의 캐시 서버 플랫폼을 합리적인 가격으로 서비스에 활용할 수 있습니다.

  • HTTP/2 및 IPv6 기술 지원(예정)

    전통적인 웹 표준 프로토콜인 HTTP1.0과 HTTP1.1은 물론, 2015년 상반기 표준화 IETF에 의해 제정된 HTTP/2 프로토콜을 지원합니다. 현재 사용되고 있는 IP 주소 체계인 IPv4의 단점을 개선하기 위해 새롭게 개발된 IPv6도 지원합니다.

이용 신청하기

상세기능

빠르고 안정적인 콘텐츠 전송이 가능한 고객의 글로벌 서비스를 위해 아래의 기술 및 기능을 제공합니다.
(Global CDN은 해외 거점을 기반으로 한 상품이므로, 국내에서 서비스를 제공해야 할 경우에는 CDN 상품을 이용해주시기 바랍니다.)

편리한 설정

웹 기반의 콘솔에서 캐싱에 대한 다양한 옵션을 선택하여 서비스에 꼭 맞는 구성으로 설정할 수 있습니다.
원본 서버에서 ‘Cache-Control’ 헤더를 통해 콘텐츠와 확장자 별로 캐시 만료 시간과 TTL(Time To Live) 설정이 가능합니다.

다양한 원본 서버 지원

Global CDN을 통해 콘텐츠를 전송하기 위해서는 원본 서버로 접근하여 콘텐츠를 캐시하고 전송해야 합니다. Global CDN은 HTTP/HTTPS 프로토콜을 통해 다양한 원본 서버와의 콘텐츠 연동을 지원합니다. 원본 서버로 고객이 보유 중인 다양한 웹 서버 및 네이버 클라우드 플랫폼의 File Storage, Server를 설정할 수 있으며, 연결을 위한 포트를 자유롭게 지정할 수 있습니다.
· 주의사항: HTTPS 프로토콜 사용 시 반드시 원본 서버와 HTTPS 프로토콜 통신이 필요하며, 직접 운영하는 웹 서버를 Global CDN의 원본 서버로 사용하고자 한다면 반드시 외부와 통신할 수 있는 Public 망에서 운영 중이어야 합니다. 현재 File Storage는 HTTP만 지원하며 HTTPS는 추후 지원 예정입니다.

강력한 콘텐츠 전송 보안

많은 브라우저에서 웹 서비스를 HTTPS로 할 경우, 콘텐츠가 HTTP로 호출되면 보안 경고 문구를 보여주고 있습니다. Global CDN에서는 HTTPS 프로토콜로 콘텐츠 전송이 가능합니다. HTTP도 물론 가능하며, 2가지를 혼용하거나 HTTPS만 활용할 수 있도록 HTTP를 제약하는 것도 가능합니다. HTTPS 프로토콜을 선택할 경우, 원본 서버와도 HTTPS로 통신하여 전체적인 콘텐츠 전송 경로의 보안이 강화됩니다. 레퍼런스로 등록된 도메인을 기준으로 하여 접근을 허용하는 기능을 제공하며, Secure-Token 기반 보안 URL 생성 기능도 지원할 예정입니다.

고속 Purge 기능

동일한 콘텐츠명으로 원본 콘텐츠를 변경할 때, 전 세계에 분산된 캐시 서버에 이를 즉각적으로 반영할 수 있도록 고속 Purge 기능을 제공합니다. Purge를 수행하여 전 세계 수십만 대의 캐시 서버에 저장되어 있는 콘텐츠를 수 초 이내에 삭제합니다. 캐시 삭제 후, 사용자 요청에 의해 새로운 콘텐츠를 원본으로부터 가져와 캐시하여 제공합니다. 이는 콘텐츠의 신뢰성을 보장하기 위한 핵심적인 기술입니다.
