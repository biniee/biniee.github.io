Backup

백업을 통해 데이터 손실을 최소화할 수 있습니다

이용 문의하기
  • 특징
  • 상세 기능
  • 요금
  • 이용안내

특징

중요한 데이터를 가장 안전한 방법으로 백업

검증된 백업 솔루션을 사용해 서버의 데이터를 정기적으로 백업하고 보관하는 서비스입니다.
유사 시 데이터 복구가 가능해 고객의 비즈니스 연속성을 보장합니다.
중요한 데이터를 안전하게 보호할 수 있도록 높은 안정성과 가용성을 기반으로 제공됩니다.

  • 다양한 플랫폼 및 데이터베이스 지원

    다양한 버전의 운영 체제(Linux 및 Windows 계열)를 지원하며 각종 데이터베이스의 온라인 백업이 가능합니다. 또한 데이터를 복구할 때 동일 계열의 운영 체제라면 원본과 다른 서버에서도 백업된 데이터를 간편하게 확인할 수 있습니다. 단, 데이터베이스의 경우 복구 대상인 서버의 환경이 원본 서버와 동일해야 합니다.

  • 높은 안정성 보장

    신뢰받고 있는 백업 솔루션을 이용하여 높은 안정성과 가용성을 보장합니다. 전문 인력이 고객의 백업 업무를 대행하여 안전하게 데이터를 백업하며, 백업 결과는 일별 리포트를 통해 제공됩니다. (2017년 하반기에는 지리적으로 분리된 멀티 존 간의 백업 데이터를 이중화하는 기능이 추가될 예정입니다.)

  • 간편한 이용

    간단한 설치 스크립트 수행만으로 백업 Agent의 설치부터 구성까지 완료할 수 있어 간편하게 백업 서비스를 이용할 수 있습니다. 서버는 파일 단위부터 디렉토리 단위까지, 데이터베이스의 경우는 테이블 단위부터 전체 DB 단위까지 백업 대상으로 선택할 수 있습니다. 데이터베이스 온라인 백업은 별도의 백업 저장 장치로 데이터가 저장되므로 서버에서 백업을 위한 추가적인 볼륨 용량을 따로 관리할 필요가 없습니다.

  • 효율적인 비용 관리

    백업 대상의 데이터 전체를 백업하는 방식과 변경된 데이터만 추출하여 백업하는 방식을 조합하여 운영할 수 있어 백업 용량을 효과적으로 관리할 수 있습니다. 백업된 데이터는 최소 1주부터 최대 24주까지 보관이 가능하며, 고객이 필요에 따라 적절한 보관 기간을 선택할 수 있어 효율적인 비용 관리가 가능합니다. 또한 백업을 위한 별도의 저장 공간이 필요하지 않아 경제적입니다.

이용 문의하기

상세기능

데이터 손실을 방지하는 것을 최우선으로 하여 설계된 네이버 클라우드 플랫폼의 Backup은
고객의 중요 데이터를 가장 안전하게 백업하고 보관해주는 상품입니다.

유연한 백업 정책

최소 1일 1회부터 최대 4주 1회까지, 백업 수행 주기를 필요에 따라 선택할 수 있습니다. 백업 데이터 보관 기관은 백업 수행 주기를 기준으로 최소 2배 이상의 기간으로 계획하여, 최소 7일부터 최대 24주 기간 내에서 자유롭게 선택 가능합니다. 백업 용량은 총 저장량 기준으로 계산되며 최소 100GB에서 수 십 TB까지 유연하게 사용할 수 있습니다. 단일 서버의 일간 백업 최대 용량은 1TB까지 지원합니다.

강력한 보안

데이터를 안전하게 백업하기 위해 백업 솔루션 내에 별도의 방화벽 기능을 적용했습니다. 방화벽 설정은 백업 Agent 설치를 위한 스크립트 수행 시 자동으로 함께 설정됩니다.
추가적인 고객의 요청에 따라 백업 스케줄 단위의 데이터 암호화를 지원해 보다 강력하게 데이터를 보호할 수 있습니다. (CAST-128, AES-256 or CAST 256 암호화 지원)

소산 백업 (이중화 백업)

로컬 존(Zone)에서 백업한 데이터를 지리적으로 떨어진 다른 존으로 이중화하여 고객의 중요 데이터를 더욱 안정적으로 보관할 수 있습니다.
이중화된 백업 데이터의 보관 주기는 기본적으로 로컬 존의 백업 데이터 주기와 동일하나, 고객의 요청에 따라 변경할 수 있습니다. (2017년 하반기 제공 예정)

백업 서비스 유형

· 파일 시스템 백업 : 네이버 클라우드 플랫폼에서 제공하는 서버(VM)의 데이터 백업 (파일 혹은 디렉토리 단위로 백업 가능)
· 데이터베이스 온라인 백업 : 네이버 클라우드 플랫폼에서 서버 이미지로 제공하는 버전의 MSSQL, MySQL의 온라인 백업 (데이터 복구를 위해 검증된 버전에 대한 지원만 제공하고 있으나, 향후 다양한 DBMS 버전을 지원할 예정입니다)

이용 문의하기

연동 부가 서비스

  • Server

    네이버 클라우드 플랫폼 내에 있는 서버를 대상으로 백업 서비스 제공

  • Multi Zone

    로컬 존에서 백업된 데이터를 지리적으로 떨어진 다른 존
    이중화(소산)하여 보관 (2017년 하반기 예정)

요금

최소 100GB에서 최대 수 십 TB까지 사용자가 원하는 데이터를 백업하고 보관합니다.
일 또는 주 단위로 백업 서비스를 이용할 수 있습니다.

서비스 요금표

서비스 요금표
구분 종량 요금제
기본 제공 용량 100GB 당 30,000원
추가 용량 1GB 당 100GB이상 ~ 1TB 이하 300원
1TB 초과 ~ 5TB 이하 250원
5TB 초과 200원
  • Backup 서비스는 최소 100GB의 데이터를 저장합니다. 실제 사용량이 100GB 이하로 저장할 경우엔 100GB 요금이 부과되며, 100GB를 초과하면 추가 사용량에 대해서 GB 단위로 과금됩니다.
  • (예) 3월 20일에 백업 서비스를 신청하고, 3월 총 저장량이 20GB인 경우 요금은 30,000원(기본 제공 용량 100GB)입니다.
  • (예) 백업 데이터 저장량이 300GB인 경우 요금은 90,000원(30,000원 + 300원*200GB)입니다.
