특징

빠르고 안정적인 Redis가 설치된 서버 제공

Redis는 보편적인 메인 메모리 기반 Key-Value 스토리지로 네이버와 LINE에서도 서비스 성능 향상을 위해 폭넓게 사용되고 있습니다.
네이버 클라우드 플랫폼에서는 추가적인 시간과 리소스 투입 없이 곧바로 Redis를 사용할 수 있도록 Redis가 설치된 서버를 제공합니다.

  • 편리한 사용

    CentOS, Ubuntu 기반에 최신 Redis 3.2가 설치된 서버를 제공하고 있습니다. 사용자는 별도의 작업 또는 어떠한 설정도 할 필요 없이 OS 이미지 생성만으로 즉시 Redis를 사용할 수 있습니다.

  • 네트워크 기반의 보안 강화

    Redis는 자체적으로 보안기능을 지원하지 않습니다. 하지만 네이버 클라우드 플랫폼에서 무료로 제공하는 네트워크 보안 기능인 ACG를 기반으로 한 강력한 네트워크 보안을 Redis 상품에 적용하여 사용할 수 있습니다.

  • 빠른 응답 성능

    데이터를 HDD 또는 SSD에 저장하는 기존 RDBMS와 달리 메인 메모리에 데이터를 저장하는 Key –Value 스토리지인 Redis는 빠른 응답 속도를 자랑 합니다. 실시간으로 응답이 필요한 서비스나 RDBMS 기반 서비스의 성능을 더욱 향상시키기 위한 캐시 용도로 사용할 경우 Redis의 장점을 극대화 할 수 있습니다.

이용 신청하기

상세기능

다양한 개발 환경 지원

Redis는 Java, C, PHP, Node js, Ruby, R, Go 등 다양한 언어를 지원하고 있으며, 수많은 오픈 소스 클라이언트를 이용하여 개발에 빠르게 적용할 수 있습니다. 더 자세한 내용은 다음 링크를 통해 확인할 수 있습니다. (https://Redis.io/clients)

다수의 자료형 제공

단순한 key Value 1:1 매핑만이 아니라 문자열의 순서를 그대로 보관할 수 있는 Lists, Value의 정수 값에 따라 언제나 정렬을 보장하는 Sorted Set,
필드와 값의 목록을 저장하기 위한 Hashes 등 다양한 자료형을 제공함으로써 비즈니스에 최적화된 데이터 구조 설계가 가능 합니다

데이터 지속성 지원

캐시 형태로 메모리에서만 사용하는 휘발성 데이터로 사용할 수 있을 뿐만 아니라 특정 시점의 데이터를 디스크에 저장하는 스냅샷 기능과 데이터가 변경될 때마다 디스크에 저장하는 AOF(Append Only File) 시스템이 지원되어 장애 발생 시 데이터 복구가 가능한 인 메모리 데이터베이스와 같은 형태로 사용이 가능 합니다.

Redis 클러스터 기반의 고가용성 지원 (지원 예정)

Redis 클러스터는 여러개의 Redis 노드들이 하나의 클러스터를 구성하는 방식으로, 각 노드로 데이터를 균등하게 분산함으로써 메인 메모리 기반의
대용량 구성이 가능합니다. 동적으로 클러스터 노드 추가 및 삭제가 가능하며(dynamic scale out) 노드에 슬레이브를 추가하여 고가용성을 지원 합니다.
