특징

손쉬운 도메인 관리로 서비스 접근성 향상 가능

DNS(Domain Name System)는 인터넷상의 도메인 이름을 실제로 접속 가능한 주소로 식별해서 찾을 수 있도록 하는 서비스입니다.
예를 들어 'http://www.example.com'라는 URL을 웹 브라우저에서 입력하면 인터넷상에서 'www.example.com'에 해당하는
실제 IP 주소를 DNS 서비스가 전달하고, 시스템은 전달된 IP 주소로 원하는 서비스에 접속할 수 있게 됩니다.
DNS 서비스는 사용자가 자체 DNS 서버를 손쉽게 구축하고 안정적인 서비스를 운영할 수 있게 합니다.
다양한 리소스 레코드를 지원하며, 편리하게 검색, 등록, 수정할 수 있습니다. 사용자는 DNS 상품을 이용해 신규 도메인을 등록하거나
네이버 클라우드 플랫폼의 DNS로 이관할 수 있고, 하위 도메인을 등록해 사용할 수 있습니다

  • 간편한 DNS 관리

    웹 기반 콘솔상에서 도메인 추가만으로 간편하게 자체 DNS를 구축하고 관리할 수 있습니다. 별도의 DNS 인프라 구축, DNS 솔루션 설치/업데이트, DNS 서버의 용량/성능관리를 포함한 운영 관리가 필요하지 않습니다.

  • 안정적인 서비스 제공

    여러 대의 DNS 서버를 운영하여 1대의 DNS 서버에 장애가 발생하더라도 다른 DNS 서버에서 도메인 응답이 가능한 구조이기 때문에 서비스 연속성을 보장합니다. 더불어 도메인 변경 작업 시 각 DNS 서버 데이터가 빠르게 동기화되어 서비스를 안정적으로 제공할 수 있습니다.

  • 손쉬운 도메인 설정

    웹 기반 콘솔에서 몇 번의 클릭 및 입력만으로 도메인 등록/변경관리가 가능합니다. IPv4(A), IPv6(AAAA), TXT, MX, NS, 역방향 조회(PTR), 메일 서버 등록 정책(SPF) 등 다양한 리소스 레코드를 설정할 수 있으며, 검색 기능 지원을 통하여 해당 zone의 리소스 레코드도 빠르게 확인할 수 있습니다.

이용 신청하기

상세기능

DNS 서비스는 도메인을 소유하고 있지만 자체 DNS 서버가 없거나, DNS 구축 경험이 부족한 고객에게 유용합니다

활용 사례

도메인을 신규로 등록해 DNS 서비스를 이용하는 경우

신규 도메인 영역을 생성하고, 생성된 도메인의 네임서버를 확인합니다. 도메인 생성 시 제공된 DNS의 네임서버를 도메인 등록 기관이나 대행 기관(가비아, KINX 등)에 사용자 도메인의 네임서버로 등록합니다(참고: 네이버 클라우드 플랫폼의 DNS 서비스는 도메인 등록을 대행하는 서비스가 아니므로 도메인 등록 기관이나 등록 대행 기관을 통해 도메인을 등록해야 합니다).

타사 또는 자체 DNS 서비스를 이용하다가 네이버 클라우드 플랫폼 DNS로 이관하는 경우

네이버 클라우드 플랫폼의 DNS에 신규 도메인 영역을 생성합니다. 기존 DNS 관리 시스템이나 자체 DNS 관리 시스템에서 현재 등록해 사용 중인 리소스 레코드 정보를 네이버 클라우드 플랫폼의 DNS 리소스 레코드 영역으로 이관(복제)합니다. 운영 중인 도메인의 네임서버 정보를 네이버 클라우드 플랫폼 DNS에서 제공하는 네임서버 정보로 변경합니다. 이때 네임서버는 도메인 등록 기관이나 대행 기관(가비아, KINX 등)에 사용자 도메인의 네임서버로 등록해야 하며, 네임서버 변경은 최대 48시간이 소요될 수 있습니다.

네이버 클라우드 플랫폼 DNS에서 하위 도메인을 운영하는 경우

하위 도메인을 신규로 생성하고, 생성한 하위 도메인의 리소스 레코드 정보를 입력합니다. 기존에 DNS를 관리하던 하위 도메인을 이관하는 경우에는 기존 리소스 레코드를 이관(복제)합니다. 상위 도메인의 리소스 레코드 설정에서 하위 도메인의 네임서버를 네이버 클라우드 플랫폼 DNS의 네임서버로 등록합니다.

기타 지표

리소스 레코드

리소스 레코드 표
레코드 내용
A 레코드 도메인 이름으로 IP 주소를 찾을 수 있도록 도메인 이름에 해당하는 IP 주소를 지정합니다. 소유 도메인 및 그 하위 도메인의 IP 주소(IPv4)를 지정할 수 있습니다.
NS(네임서버) 레코드 소유 도메인 이름의 DNS 서버(네임서버)를 지정합니다. 보통 하위 도메인 이름을 다른 DNS 서버가 관리하도록 위임(delegation)할 때 사용합니다.
AAAA(IPv6) 레코드 도메인 이름으로 IPv6 주소를 찾을 수 있도록 도메인 이름에 해당하는 IPv6 주소를 지정합니다. 소유 도메인 및 그 하위 도메인의 IPv6 주소를 지정할 수 있습니다.
CNAME(도메인 별명) 레코드 도메인 이름에 대한 별명(alias)을 지정하는 레코드입니다. 예를 들어 'server2'라는 정규(canonical) 도메인 이름의 서버가 'www' 서버로도 사용된다면, 'server2'의 별명으로 'www'를 지정합니다.
MX(메일 서버) 레코드 사용자가 보유한 도메인과 하위 도메인의 메일 서버를 지정할 수 있습니다.
PTR(역방향 조회) 레코드 IP 주소로 해당 도메인 이름을 찾을 수 있도록 IP 주소에 해당하는 도메인 이름을 지정합니다. A 레코드와 반대되는 개념입니다.
SPF(스팸 처리 방지) 레코드 보안 정책을 지정합니다. 주로 스팸 메일 방지를 위해 스팸 메일을 송신한 서버의 IP 주소 등을 명시해서 메일 주소를 사칭하지 못하게 합니다.
TXT(도메인 설명) 레코드 도메인에 대해서 텍스트 정보를 설정합니다. 주로 SPF 레코드와 같은 용도로 사용합니다.
